# SQLAlchemy Challenge - Hawaii Climate Analysis  
  
## Overview  
This project analyzes Hawaii's climate data using SQLAlchemy ORM, Pandas, and Flask. The data is ingested from CSV files and stored in a SQLite database, and a Flask API exposes various endpoints for data exploration. A separate Jupyter Notebook is provided for additional data analysis and visualization.  
  
## Files  
- **app_solved.py**: Flask application with the following API endpoints:  
  - `/api/v1.0/precipitation`: Returns JSON of precipitation data by date.  
  - `/api/v1.0/stations`: Returns JSON list of weather station IDs.  
  - `/api/v1.0/tobs`: Returns JSON of temperature observations for the most active station.  
  - `/api/v1.0/temp/<start>`: Returns JSON of min, avg, and max temperature for dates starting from `<start>`.  
  - `/api/v1.0/temp/<start>/<end>`: Returns JSON of min, avg, and max temperature for the date range `<start>` to `<end>`.  
  
- **populate_db.py**: Script to create and populate the SQLite database (`hawaii.sqlite`) from CSV files available in the **Resources** folder.  
  
- **hawaii.sqlite**: SQLite database generated by `populate_db.py` containing two tables:  
  - **measurement** (composite primary key on `(station, date)`)  
  - **station** (primary key on `station`)  
  
- **Resources/**: Directory containing the source CSV files:  
  - `hawaii_measurements.csv`  
  - `hawaii_stations.csv`  
  
- **climate_analysis.ipynb** (or `climate_starter.ipynb`): Jupyter Notebook for further exploration and visualization of the climate data. This file is optional and provided for deeper analysis beyond the API.  
  
## Usage  
  
1. **Database Setup:**  
   - Place `hawaii_measurements.csv` and `hawaii_stations.csv` in the `Resources` folder.  
   - Run `populate_db.py` to create and populate the `hawaii.sqlite` database.  
  
2. **Run the Flask Application:**  
   - Execute `app_solved.py` using Python to start the Flask server.  
   - Navigate to `http://127.0.0.1:5000/` in your web browser to view available API endpoints.  
  
3. **API Testing:**  
   - Use a browser or tools like `curl` or Postman to test the endpoints listed above.  
  
4. **Additional Analysis:**  
   - Open `climate_analysis.ipynb` (or `climate_starter.ipynb`) in Jupyter Notebook to view additional data analysis and visualizations of the climate data.  
  
Enjoy exploring Hawaii's climate data!
